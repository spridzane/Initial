18. exercise: Git branches and merging tools

  1. Review presentation

      

  2. If necessary, install meld and diffuse tools:

    sudo apt install meld diffuse

  3. Open your project:

    cd ~/workspace/Aaaa

    where Aaaa is code of your project.

  4. Add new remote repository https://odo.lv/git/JTM/ to your project:

    git remote add upstream https://odo.lv/git/JTM/
  5. Add, commit and push changes of your project to the bcserver (to be sure you have all your previous work on bcserver).

  6. Pull branches branch1 and branch2 from upstream repository to branches into your repository:

    git fetch upstream branch1:branch1
    git fetch upstream branch2:branch2

  7. Check that your repository has three branches:

    git branch

    returns:

      branch1
      branch2
    * master
  8. Make two copies of your project, for example Aaaa1 and Aaaa2 as sibling projects in your workspace.

  9. Check out branches in copies of the project:

    cd ~/workspace/Aaaa1
    git checkout branch1
    cd ~/workspace/Aaaa2
    git checkout branch2
  10. Open meld, and compare content of your original project, e.g. Aaaa and Aaaa1, copy and save missing parts of content into Aaaa project.
  11. Compare content of catbN.txt files in jtm.activity18 package between projects with master and/or branch1/branch2 branches.
  12. Merge content of all numbered filees catbN.txt into single catb.txt file
  13. Check that catb.txt file has proper content by running GitMergeTest1.java unit test.

  14. Before you add, commit and push changes to bcserver, ensure you are working on master branch, e.g. command:

    git branch

    returns:

      branch1
      branch2
    * master
